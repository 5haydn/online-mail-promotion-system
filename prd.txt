# 在线邮件推广系统设计方案（B/S 架构 + 火山云部署 + 内置豆包大模型）

## 一、项目概述

### 1.1 系统简介

本系统采用**浏览器 / 服务器（B/S）架构**，用户通过浏览器访问系统，无需安装客户端，统一部署在火山引擎云平台；深度集成豆包大模型，支持 Excel 客户信息批量上传、AI 自动个性化生成推广邮件、批量邮件发送、IMAP 轮询监听客户邮件回复，一旦检测到回复立即通过站内消息、浏览器推送、钉钉 / 企业微信推送通知运营人员，实现邮件营销全流程线上化、智能化。

### 1.2 核心业务流程

1. 管理员 / 运营人员浏览器登录系统
2. 下载标准模板 → 上传客户 Excel 表格 → 数据校验、清洗入库
3. 配置邮件推广规则、预设卖点、发送邮箱账号
4. 调用豆包大模型，基于客户信息批量生成个性化邮件主题 + 正文
5. 预览、编辑 AI 草稿，配置发送时间、发送频率，批量发送邮件
6. 服务端定时任务通过 IMAP 拉取收件箱，自动检测客户回复邮件
7. 检测到新回复后，实时推送站内通知 + 浏览器消息提醒，支持在线查看回复内容

## 二、技术架构选型（B/S）

### 2.1 前后端技术栈

表格

| 分层 | 技术选型 | 说明 |
| --- | --- | --- |
| 前端（浏览器端） | Vue3 + Element Plus + Axios | 管理后台页面、Excel 上传、数据预览、邮件编辑、消息通知、统计看板 |
| 后端服务 | SpringBoot 3.x / FastAPI | 业务接口、文件解析、豆包 API 封装、邮件发送、定时任务、消息推送 |
| 数据库 | 火山云 RDS MySQL 8.0 | 用户、客户数据、邮件任务、发送记录、回复记录、模板配置 |
| 缓存 & 消息队列 | 火山云 Redis | 接口限流、在线用户、任务状态缓存、消息临时存储、异步任务削峰 |
| 对象存储 | 火山云 TOS | 存储上传 Excel 文件、错误数据文件、邮件附件、AI 生成日志 |
| AI 能力 | 豆包开放平台 API | 文案生成、内容润色、个性化邮件创作、违规内容检测 |
| 邮件服务 | SMTP（发信）+ IMAP（收信监听） | 企业邮箱 / 第三方邮件推送服务，批量发信 + 回复轮询拉取 |
| 实时推送 | WebSocket + 浏览器 Notification | 新回复实时站内推送、浏览器桌面弹窗通知 |

### 2.2 火山云部署架构

1. **计算资源：火山云 ECS**
部署后端服务、定时任务服务（邮件回复轮询、异步发送任务），配置弹性公网 IP，安全组开放 80/443 端口。
2. **数据库：火山云 RDS MySQL + 火山云 Redis**
云托管数据库，自动备份、高可用，避免自建数据库运维风险。
3. **存储：火山云 TOS 对象存储**
存放所有用户上传的 Excel 文件、附件、导出报表，设置访问权限，防盗链。
4. **CDN（可选）**
静态资源加速，提升后台页面访问速度。
5. **运维配套**
火山云云监控：服务器 CPU、内存、接口异常告警；快照定期备份；日志服务收集操作日志、发送日志。

## 三、系统核心功能模块设计

### 模块 1：权限与用户管理

1. 账号登录、验证码登录、密码重置
2. 角色权限：超级管理员、运营人员、只读查看员
3. 功能权限控制：文件上传、邮件发送、AI 配置、API 密钥管理、数据导出
4. 操作日志：记录登录、文件上传、批量发信、AI 文案生成等关键操作

### 模块 2：Excel 客户数据管理

1. 提供标准 Excel 模板下载（字段：客户邮箱、姓名、公司、行业、意向产品、联系方式、备注）
2. 支持 xlsx/xls 文件上传，文件先上传至火山云 TOS，后端异步解析表格
3. 数据校验：邮箱格式校验、空值过滤、重复邮箱去重、非法数据标记
4. 校验失败数据单独生成错误 Excel 包，支持重新下载修正后二次上传
5. 客户分组、标签管理，可按行业、意向产品筛选批量创建邮件任务

### 模块 3：豆包 AI 智能邮件文案生成（核心）

1. 系统后台统一配置豆包 API 密钥，管理员权限管控，前端不可直接暴露密钥
2. 两种文案生成模式：
   - 统一模板生成：所有客户使用同一套 AI 生成的邮件主题 + 正文
   - 个性化生成：读取 Excel 中客户公司、行业、意向产品等字段，动态传入 Prompt，豆包为每个客户生成专属推广邮件
3. 支持自定义 Prompt 配置：商务正式 / 亲切营销 / 简洁版三种语气、可设置邮件结尾落款、禁止违规营销话术
4. AI 内容安全校验：过滤敏感词、违规营销内容，防止邮件被邮箱服务商拦截
5. 生成后在线预览、批量编辑、单条修改、保存为自定义邮件模板

### 模块 4：邮件批量发送管理

1. 多发件邮箱账号配置（SMTP 配置、别名、发送频率限制），自动轮询发件账号避免 IP 风控
2. 发送策略：立即发送、定时发送、间隔限流发送（防止短时间大量发信被封禁）
3. 支持添加附件、邮件 HTML 富文本编辑
4. 实时展示发送进度：成功数、失败数、失败原因（邮箱不存在、被退信、服务器拒绝）
5. 发送记录持久化存储，可筛选、导出发送报表

### 模块 5：邮件回复自动监听与消息推送

1. 后台定时任务（基于 Redis 延时任务 / Quartz），通过 IMAP 协议定时拉取配置的收件邮箱
2. 自动关联原有发送任务，匹配客户邮箱，判断是否为本次推广邮件的回复
3. 识别回复类型：咨询、拒绝、退信、意向沟通等，可交由豆包对回复内容做摘要解析
4. 实时推送方案：
   - WebSocket 站内消息：后台消息中心红点提醒
   - 浏览器桌面通知（Notification）弹窗提醒
   - 可选扩展：钉钉机器人 / 企业微信应用推送消息
5. 在线查看回复原文、AI 摘要，支持标记跟进状态（待处理、已跟进、无意向）

### 模块 6：消息中心与数据统计看板

1. 消息中心：所有新回复通知统一汇总，已读 / 未读筛选
2. 数据大屏：总导入客户数、AI 生成邮件数量、发送成功率、回复率、意向客户统计
3. 多维度报表：按时间、客户分组、发件账号统计营销效果，支持 Excel/PDF 导出

### 模块 7：系统配置管理

1. 豆包大模型参数配置：API 密钥、模型选择、温度、最大上下文、Prompt 模板预设
2. SMTP/IMAP 邮箱账号配置、发送限流规则
3. 推送渠道配置：浏览器推送、钉钉 / 企业微信推送参数
4. 文件上传大小限制、TOS 存储路径配置、数据备份策略

## 四、核心业务时序图（关键流程）

1. 运营浏览器上传 Excel → 文件存入火山云 TOS → 后端解析清洗数据入库
2. 用户设置推广需求、选择客户分组 → 前端提交 AI 生成请求
3. 后端携带客户字段 + 自定义 Prompt 调用豆包开放 API，批量生成邮件内容
4. 用户在线编辑确认文案，配置 SMTP 发件账号与发送策略
5. 后端异步任务批量发送邮件，记录每一条发送结果
6. 定时任务周期性 IMAP 拉取收件箱，匹配客户邮箱识别回复
7. 新回复写入数据库，通过 WebSocket 向前端推送实时通知，浏览器弹出提醒
8. 运营在消息中心查看回复，可利用豆包生成回复话术跟进客户

## 五、安全设计（火山云 + 应用层）

1. 接口权限：基于 Token 鉴权，接口防越权访问，敏感配置仅管理员可见
2. 密钥安全：豆包 API 密钥、邮箱账号密码加密存储在 MySQL，前端永不明文返回
3. 火山云安全组限制出站入站 IP，TOS 文件设置私有读权限，防止客户数据泄露
4. 上传文件类型白名单，禁止可执行文件上传，防止恶意脚本攻击
5. AI 生成内容敏感词校验、邮件内容合规检测，规避营销违规风险
6. 操作日志全留存，支持安全审计与问题溯源

## 六、部署步骤简要说明

1. 火山云开通 ECS、RDS MySQL、Redis、TOS 对象存储服务，配置 VPC 内网互通
2. 安全组开放 80/443，配置域名 + SSL 证书实现 HTTPS 访问
3. 后端服务打包部署至 ECS，配置数据库、Redis、TOS 连接参数
4. 管理员登录后台，配置豆包开放平台 API 密钥、SMTP/IMAP 邮箱账号
5. 前端 Vue 打包静态资源部署至 ECS 或火山云 CDN，绑定域名对外访问
6. 配置定时任务、WebSocket 服务，开启邮件回复监听与消息推送功能
